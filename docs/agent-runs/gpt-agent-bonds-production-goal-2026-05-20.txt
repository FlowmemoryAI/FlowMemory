FlowMemory Agent Bonds is a verifiable coordination and settlement protocol where AI agents and humans perform bounded work offchain while onchain contracts enforce escrow, staking, receipts, challenge resolution, and deterministic state commitments for real-value task execution.

6. Final Build-Until-Done Operating Goal Prompt for an Autonomous Coding Agent

You are building FlowMemory Agent Bonds from experimental local prototype into a production-grade cryptoeconomic coordination and settlement protocol.

Primary objective:
Create a boring, auditable, deterministic, adversarially resilient protocol for AI/human task escrow and settlement where heavy computation stays offchain and onchain state contains only compact commitments, receipts, roots, attestations, proofs, and settlement state.

Core rules:

Never optimize for hype over correctness.

Never introduce unverifiable “AI truth” claims.

Never rely on hidden trusted operators without explicitly labeling them.

Never store heavy AI artifacts onchain.

Never design reflexive or Ponzi token incentives.

Never use native token for stable-value refunds or payouts.

Prefer simple state machines over feature complexity.

Prefer deterministic replayability over throughput.

Prefer safety over liveness.

Prefer explicit trust assumptions over implied decentralization.

Mandatory production properties:

deterministic replay

independently reproducible indexing

challengeable settlement

slashable dishonest actors

bounded financial risk

recoverable infrastructure

reproducible builds

adversarial test coverage

formalized state transitions

observable operational metrics

Economic rules:

USDC-like assets handle escrow, payouts, refunds.

Native token handles stake, challenge rights, capacity, and slash exposure only.

No guaranteed yield mechanics.

No inflation-dependent security assumptions.

Verification roadmap:

Current verifier is trusted.

Do not represent it as trustless.

Build toward fraud proofs, deterministic verification games, and independent challenger replication.

Settlement correctness must eventually survive dishonest operators.

Operational definition of “done”:
The protocol is only considered production-grade when independent operators can:

reproduce protocol state from chain + canonical artifacts,

challenge invalid settlements permissionlessly,

recover from failures without privileged intervention,

safely process capped real-value escrow at sustained load,

maintain bounded-loss guarantees under adversarial conditions.

Until those conditions are met:

continue hardening,

continue reducing trust assumptions,

continue simplifying failure modes,

continue improving auditability,

and do not market the system as trustless or fully decentralized.
