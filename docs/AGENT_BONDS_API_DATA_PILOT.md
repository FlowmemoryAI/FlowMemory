# Agent Bonds API/Data Pilot Template

## Why this wedge

The first strong commercial wedge is objective API/data work.

Reasons:
- narrow task boundaries;
- clear schemas;
- repeatable verifier checks;
- measurable failures;
- easier recourse pricing than open-ended code or research.

## Pilot shape

The current template models:
- `data.extract`
- `api.call`

Each task envelope includes:
- schema-valid acceptance criteria;
- source/API receipt requirements;
- evidence commitment rules;
- x402 escrow-bridge funding metadata;
- optional recourse compatibility.

## Not in this pilot

- open-ended research
- production deployment tasks
- subjective human-quality deliverables
- broad autonomous multi-agent workflows
